# 2026-07-02 GC 模式对比基准

本报告来自本机 Windows 11 / AMD Ryzen 7 5800X / .NET 10.0.8，覆盖 plan/16 §4.6 的 Workstation+Concurrent 与 Server+Concurrent 对比。两组基准都在 `GcPauseBenchmark.GlobalSetup` 中切到 `GCSettings.LatencyMode = SustainedLowLatency`，并在 `GlobalCleanup` 恢复原模式。命令使用 BenchmarkDotNet short job、`WarmupCount=1`、`IterationCount=3`，用于当前 M0/M10 性能纪律定档证据；最终发布前仍应以目标硬件正式长跑替换。

Workstation+Concurrent：

```pwsh
dotnet run --project bench/PixelEngine.Benchmarks/PixelEngine.Benchmarks.csproj -c Release --no-build -- --filter *GcPauseBenchmark.SteadySimulationTick* --artifacts artifacts/gc-workstation --job short --warmupCount 1 --iterationCount 3
```

Server+Concurrent：

```pwsh
$env:DOTNET_gcServer='1'; $env:DOTNET_GCServer='1'; $env:COMPlus_gcServer='1'; dotnet run --project bench/PixelEngine.Benchmarks/PixelEngine.Benchmarks.csproj -c Release --no-build -- --filter *GcPauseBenchmark.SteadySimulationTick* --artifacts artifacts/gc-server --job short --warmupCount 1 --iterationCount 3
```

| 模式 | Runtime 报告 | Mean | Min | Median | Max | GC | Allocated |
|---|---|---:|---:|---:|---:|---|---|
| Workstation+Concurrent + SustainedLowLatency | `GC=Concurrent Workstation` | 2.179 us | 2.079 us | 2.148 us | 2.310 us | 0 / 0 / 0 | 0 B/op |
| Server+Concurrent + SustainedLowLatency | `GC=Concurrent Server` | 2.208 us | 2.104 us | 2.228 us | 2.292 us | 0 / 0 / 0 | 0 B/op |

本次稳态 tick 在两种 GC 模式下均未触发 Gen0/Gen1/Gen2，`MemoryDiagnoser` 报告零托管分配。Workstation 均值略低，Server 最大值略低，但差距处于短基准噪声范围内，当前没有足够证据要求强制切 Server GC。因此当前定档为保留运行时默认 Workstation+Concurrent，并由 `EngineBuilder` 与 benchmark 统一切到 `SustainedLowLatency`。后续若大 pinned 堆、长时间世界流式或发布目标硬件显示 Server GC 最坏停顿更低，再以正式长跑更新定档。
