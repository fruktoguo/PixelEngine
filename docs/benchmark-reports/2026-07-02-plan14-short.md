# 2026-07-02 plan/14 BenchmarkDotNet Short 校准

本报告来自本机 Windows 11 / AMD Ryzen 7 5800X / .NET 10.0.8，命令使用 BenchmarkDotNet `Short` job、`WarmupCount=1`、`IterationCount=1`，只用于验证 plan/14 基准可运行、报告可产出、并为 §1.4 / §12.7 / §12.8 提供当前实现的短跑校准点。该数据不是最终目标硬件统计基线，不能替代正式多迭代 benchmark。

> JobSystem `ParallelRange` 派发分配数字已被 `docs/benchmark-reports/2026-07-03-jobsystem-parallelrange-zero-allocation.md` 取代；该新报告确认 `ParallelRange` 与 `ParallelRangeRaw` 多 worker 稳态派发均为 `Allocated 0 B`。

命令：

```pwsh
dotnet run --project bench\PixelEngine.Benchmarks\PixelEngine.Benchmarks.csproj -c Release --no-build -- --filter "*CellThroughputBenchmark.StepSingleThread*" "*CoreScalingBenchmark.ParallelRangeSum*" "*TextureUploadBenchmark.FullFramePboCopy*" "*ReactionLookupBenchmark.FindDirect*" "*GcPauseBenchmark.SteadySimulationTick*" "*ParticleIntegrationBenchmark.IntegrateFlyingParticles*" --artifacts artifacts/plan14-benchmarks --job Short --warmupCount 1 --iterationCount 1 --exporters markdown
dotnet run --project bench\PixelEngine.Benchmarks\PixelEngine.Benchmarks.csproj -c Release --no-build -- --filter "*CellThroughputBenchmark.StepJobSystem*" --artifacts artifacts/plan14-benchmarks --job Short --warmupCount 1 --iterationCount 1 --exporters markdown
```

## 摘要

| 类别 | Benchmark | 参数 | Mean | Allocated | 结论 |
|---|---|---:|---:|---:|---|
| CA cells/frame | `CellThroughputBenchmark.StepJobSystem` | `FullActiveLiquid` | 38.327 ms | 544 B | 262,144 active cells 约 6.84M cells/s，8ms 预算约 54.7K active cells，当前实现未达 §1.4 / §12.8 目标。 |
| CA dirty-rect | `CellThroughputBenchmark.StepJobSystem` | `TypicalDirtyRect` | 413.1 us | 0 B | 典型小 dirty rect 有大量余量，固定调度 overhead 主导。 |
| Core scaling | `CoreScalingBenchmark.ParallelRangeSum` | 1,048,576 items / workers 1,2,4,0 | 303.02 us / 230.69 us / 162.16 us / 183.80 us | 0-136 B | 大任务 4 worker 对 1 worker 约 1.87x；默认 worker 受调度 overhead 与本机噪声影响。 |
| Texture upload | `TextureUploadBenchmark.FullFramePboCopy` | 1920x1080 BGRA8 | 350.1 us | 0 B | 约 8.29 MiB copy，CPU 侧 memcpy 量级约 23.7 GB/s。 |
| Reaction lookup | `ReactionLookupBenchmark.FindInert/FindLinear/FindBinary/FindDirect` | 0 / 4 / 16 / 40 reactions | 1.246 ns / 4.817 ns / 3.190 ns / 2.891 ns | 0 B | 查表策略均为 ns 级且无托管分配；硬件 cache-miss counter 需管理员 ETW Kernel Session，本次普通权限未产出 Cache Misses 列。 |
| GC steady tick | `GcPauseBenchmark.SteadySimulationTick` | single dirty cell | 2.145 us | 0 B | `MemoryDiagnoser` 报告 `Allocated` 为 `-`，即 0 B/op。 |
| Particle integration | `ParticleIntegrationBenchmark.IntegrateFlyingParticles` | 50K / 100K / 200K | 473.2 us / 915.3 us / 2.088 ms | 0 B | 200K 飞行粒子约 95.8M particles/s，无托管分配。 |

## 后续判定

本次报告完成 plan/14 的 “GC 稳态 tick 零分配” 证据，并产出 cells/frame、每核加速曲线、纹理上传、反应查表延迟、GC、粒子积分六类普通 BenchmarkDotNet 报告。反应 cache-miss 硬件计数器尚未完成：启用 `PIXELENGINE_BENCH_HARDWARE_COUNTERS=1` 后，BenchmarkDotNet 在本机提示需要管理员权限的 ETW Kernel Session。full-active CA 吞吐也未达 §1.4 的 2-4M active cells / 8ms 目标。该性能缺口应进入 plan/16 性能加固：优先审计 CA full-active 热循环、JobSystem 调度开销、per-cell branch / bounds-check / dirty-rect 交互，并用正式多迭代 BenchmarkDotNet 报告替换本 Short 校准。
